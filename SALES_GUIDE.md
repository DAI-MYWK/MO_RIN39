# RecruitMO UI プロトタイプツール - 営業活用ガイド

## 📋 ツール概要

このツールは、飲食店向けモバイルオーダーシステム（RecruitMO）の導入時に、**店舗のメニューを実際の UI パターンで視覚化し、最適なデザインを選択するための営業支援ツール**です。

---

## 🎯 営業での使用場面

### 1. **初回訪問・ヒアリング時**

#### 使用タイミング

- 店舗のメニュー構成を理解したい時
- オーナーが「うちのメニューは多いから、どう見せたらいいか分からない」と悩んでいる時
- 既存のメニュー表やホームページを見せてもらった時

#### 使い方

1. **食べログやホットペッパーグルメの URL を入力**

   - 「まずは、こちらのツールで実際のメニューを表示してみましょうか」
   - 「食べログの URL があれば、すぐにメニューを取得できます」

2. **自動収集されたメニューを確認**
   - 「お店のメニューが自動で取得できました。これが実際のモバイルオーダー画面になります」

#### トーク例

```
「実は、モバイルオーダーの導入で最も重要なのは、メニューの見せ方なんです。
お客様がスマホで見た時に、どういう順番で、どういう見た目でメニューが並ぶかで、
注文数や客単価が大きく変わります。

今日は、お店のメニューを実際に4つのパターンで表示してみて、
どれが一番お店に合っているか、一緒に見てみませんか？」
```

---

### 2. **提案・デモンストレーション時**

#### 使用タイミング

- 複数の UI パターンを比較して提案したい時
- オーナーに「どれが良いか」を視覚的に判断してもらいたい時
- デザインの違いを具体的に説明したい時

#### 使い方

1. **「パターン比較」タブを開く**
2. **2 つのパターンをサイドバイサイドで表示**
   - スクロール同期機能で、同じ位置のメニューを比較可能
3. **ヒートマップ機能を ON にして、視覚的重みを可視化**
   - 「赤い部分が目立つメニューです。お客様の視線が集まりやすい場所です」

#### トーク例

```
「こちらが4つのUIパターンです。

【左：カテゴリ重視型】
カテゴリごとに整理されているので、『パスタが食べたい』というお客様には
すぐに目的のメニューにたどり着けます。メニュー数が多いお店に適しています。

【右：写真重視型】
大きな写真でメニューを魅力的に見せます。写真がきれいなお店や、
見た目で選んでもらいたいメニューがある場合に効果的です。

スクロールを動かすと、同じ位置のメニューを比較できます。
どちらがお店の雰囲気に合っていますか？」
```

---

### 3. **カスタマイズ・調整時**

#### 使用タイミング

- オーナーが「このメニューをもっと目立たせたい」と言った時
- 「おすすめメニューを強調したい」という要望がある時
- メニュー名や説明文を変更したい時

#### 使い方

1. **「編集モード」を ON**
2. **プレビュー上で直接編集**
   - メニュー名、価格、説明文をリアルタイムで変更可能
3. **変更後の見た目を即座に確認**

#### トーク例

```
「例えば、こちらの『和牛ステーキ』を、もっと目立たせたい場合。

【編集モードをON】
→ メニュー名を『【期間限定】和牛ステーキ』に変更
→ 説明文に『厳選したA5ランクの和牛を使用』を追加

すると、このように表示が変わります。お客様の目に留まりやすくなりますね。

実際のシステムでも、このように簡単に変更できます。」
```

---

### 4. **クロージング・意思決定時**

#### 使用タイミング

- オーナーが「どれにすればいいか迷っている」時
- 「実際に使ってみないと分からない」という不安がある時
- 導入の最終判断を促したい時

#### 使い方

1. **「おすすめ重視型」パターンを表示**
   - 「おすすめメニューを前面に出すことで、客単価が上がる傾向があります」
2. **ヒートマップで視覚的重みを確認**
   - 「この赤い部分が、お客様が最初に見る場所です。ここにおすすめメニューを配置すると効果的です」
3. **デバイスシミュレーションで実機感を演出**
   - 「iPhone SE と iPhone 15 Pro Max で、実際の見え方を確認できます」

#### トーク例

```
「4つのパターンを見ていただきましたが、お店の特徴を考えると、
【おすすめ重視型】が最も効果的だと思います。

理由は3つあります：

1. お店の看板メニュー『和牛ステーキ』が最初に目に入る
   → お客様が迷わず注文できる

2. ヒートマップを見ると、この位置（画面の上部）が最も目立つ
   → ここにおすすめメニューを配置すると、注文率が上がる

3. 写真が大きく表示されるので、お店のこだわりが伝わる
   → 価格を高めに設定しても、納得して注文してもらえる

実際の導入後も、このようにデータを見ながら調整できます。
まずはこのパターンで始めてみませんか？」
```

## 💡 効果的なトークのポイント

### ✅ DO（推奨）

1. **「一緒に見る」姿勢**

   - 「一緒に見てみましょうか」
   - 「どれがお店に合っているか、一緒に考えましょう」

2. **具体的な数値や効果を示す**

   - 「このパターンだと、おすすめメニューの注文率が 20%上がる傾向があります」
   - 「ヒートマップの赤い部分が、お客様が最初に見る場所です」

3. **店舗の特徴に合わせて説明**

   - 「メニュー数が多いので、カテゴリ重視型がおすすめです」
   - 「写真がきれいなので、写真重視型が効果的です」

4. **実際の操作を見せる**
   - 「編集モードを ON にすると、このように変更できます」
   - 「スクロールを動かすと、同じ位置のメニューを比較できます」

### ❌ DON'T（避けるべき）

1. **技術的な説明をしすぎる**

   - 「このツールは Next.js で作られていて...」（不要）
   - 「スクレイピングの API が...」（不要）

2. **一方的に説明する**

   - 「このパターンが良いです」（一方的）
   - 「これに決めましょう」（押し付け）

3. **抽象的な表現を使う**
   - 「きれいです」（抽象的）
   - 「使いやすいです」（抽象的）

## 📝 まとめ

このツールは、**「見た目を比較する」から「売上を最適化する」**ための営業支援ツールです。

- ✅ **視覚的に理解してもらえる**：抽象的な説明ではなく、実際の画面を見せられる
- ✅ **一緒に決められる**：オーナーと一緒に見ながら、最適なパターンを選択できる
- ✅ **不安を解消できる**：「実際に使ってみないと分からない」という不安を解消できる
- ✅ **クロージングに繋げられる**：具体的な画面を見せることで、導入の意思決定を促せる

営業の場面で効果的に活用し、より多くの店舗に RecruitMO を導入しましょう！
